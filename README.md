# BOSS海投助手

一个基于 Chrome / Edge Manifest V3 的浏览器扩展，会在 BOSS 直聘职位页右上角注入悬浮小窗，用来做职位筛选、自动逐个查看、自动尝试沟通，并在页面内实时输出运行日志。

当前版本：`0.1.1`

## 功能概览

- 右上角悬浮助手小窗
- 只从左侧职位列表读取岗位，不混入右侧详情区的杂项内容
- 点击 `开始海投` 后，先自动把职位列表滑到最下面，尽量读取完整懒加载职位
- 支持职位关键词、工作地关键词筛选
- 支持高级可选项：
  - 薪资关键词
  - 公司关键词
  - 打招呼语模板
  - 轮询间隔
  - 自动打招呼
  - 自动切下一个
  - 自动回复（可接入豆包 / DeepSeek / 腾讯元宝）
- 支持尝试 `立即沟通 -> 留在此页 -> 继续处理下一个`
- 页面内日志卡片，方便排查行为是否符合预期

## 适用范围

- 浏览器：Chrome、Edge
- 页面：`https://www.zhipin.com/*`
- 运行方式：作为“加载已解压的扩展程序”本地安装

## 安装方法

### Chrome

1. 打开 `chrome://extensions/`
2. 开启右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目根目录

### Edge

1. 打开 `edge://extensions/`
2. 开启“开发人员模式”
3. 点击“加载解压缩的扩展”
4. 选择本项目根目录

安装成功后，打开 BOSS 直聘职位页，页面右上角会出现悬浮助手窗口。

## 使用说明

### 1. 进入职位页

先进入 BOSS 直聘的职位列表页面，而不是其他频道页。扩展只有在职位页才会启用。

### 2. 填写主筛选条件

主窗口保留了两项高频条件：

- `职位名包含`
- `工作地包含`

示例：

- 职位：`后端, Java, Go`
- 地点：`上海, 浦东, 远程`

多个关键词使用中文逗号、英文逗号、空格都可以分隔。

### 3. 点击开始海投

点击 `开始海投` 后，扩展会先做一轮完整预读：

1. 自动滚动左侧职位列表到底部
2. 尽量读取完整职位数据
3. 再开始判断哪些职位命中筛选
4. 逐个点击职位卡片并尝试执行沟通动作

这一步是为了避免“页面还没读完就提示没有岗位”的误判。

### 4. 查看运行日志

主窗口底部带有日志卡片，会持续输出最近运行记录，例如：

- 已开始预加载职位
- 已选中某个职位
- 已点击立即沟通并留在此页
- 当前职位未完成，将稍后重试
- 没有新的匹配职位，继续下滚列表

如果行为不对，优先看日志。

## 高级选项

点击主窗口右上角齿轮，会打开一个单独的高级设置窗口。

可配置项包括：

- `薪资包含（可选）`
- `公司名包含（可选）`
- `打招呼语模板`
- `轮询间隔(ms)`
- `自动打招呼`
- `自动切下一个`
- `自动回复开关 / 服务商 / API Key / Model / Endpoint / 提示词 / 温度 / 回复上限`

### 打招呼语模板变量

模板支持以下变量：

- `{jobTitle}`
- `{companyName}`
- `{location}`
- `{salary}`

默认模板：

```text
您好，我对{jobTitle}岗位很感兴趣，已认真阅读职位描述，期待进一步沟通。
```

## 当前自动化行为

当前实现大致按下面的顺序执行：

1. 从左侧职位列表中提取职位
2. 根据关键词计算是否命中
3. 点击命中的职位卡片
4. 等待右侧职位详情切换完成
5. 在详情区内优先查找 `立即沟通 / 沟通 / 打招呼`
6. 如果出现 `留在此页`，点击后继续留在当前列表页
7. 如果页面进入 `继续沟通 / 已沟通` 状态，也会视为已处理
8. 如果聊天输入框出现，会尝试填充招呼语并发送

## 注意事项

1. BOSS 页面结构经常变化，扩展逻辑依赖当前 DOM 结构，后续可能需要继续适配。
2. 不同账号、不同城市、不同实验分组下，按钮样式和弹窗结构可能不同。
3. 自动化行为受页面懒加载、网络速度、账号状态影响，偶发漏点或识别失败是可能的。
4. 当前主要围绕职位列表页工作，不适合在聊天页、公司页等其他页面运行。
5. 扩展不会绕过站点本身的限制；如果页面本身不给沟通、不给投递，日志会显示失败或未识别。

## 常见问题

### 1. 看不到悬浮窗

- 确认扩展已在浏览器扩展页启用
- 确认当前页面是 BOSS 职位页
- 尝试刷新当前页面
- 尝试在扩展页重新加载本扩展

### 2. 改了代码但页面没变化

内容脚本会被浏览器缓存到当前页里。改完代码后请执行：

1. 到扩展管理页点击“刷新 / Reload”
2. 回到 BOSS 页面整页强刷一次

### 3. 一直提示没有匹配职位

- 先确认左侧职位列表里确实有符合条件的岗位
- 放宽职位关键词或地点关键词
- 如果只想按职位和地点筛选，请在高级选项里清空薪资和公司关键词

### 4. 日志说点了沟通，但页面没反应

这通常说明页面按钮结构发生变化，或者当前账号下的按钮并不是标准的 `立即沟通` 流程。需要按实际 DOM 再做适配。

## 项目结构

```text
.
├─ manifest.json
├─ advanced.html
└─ src
   ├─ background.js
   ├─ content.js
   └─ advanced.js
```

### 文件说明

- `manifest.json`：扩展声明文件
- `src/content.js`：页面悬浮窗、职位识别、滚动预读、自动流程主逻辑
- `src/background.js`：设置持久化、高级选项窗口管理
- `advanced.html`：高级设置弹窗页面
- `src/advanced.js`：高级设置读写逻辑

## 开发调试

本项目不依赖打包工具，直接修改源码即可。

每次修改后建议执行：

1. 浏览器扩展页刷新扩展
2. BOSS 页面整页刷新
3. 打开控制台和日志卡片一起看行为
